[Dehashed](https://github.com/sm00v/Dehashed) is a reconnaissance tool used to search breach data repositories for exposed credentials. It allows analysts to identify **cleartext passwords, password hashes, usernames, and associated metadata** leaked from historical data breaches.

The platform can be accessed either through the **web interface** or programmatically via its **API using custom scripts**.

⚠️ **Registration is required** to use Dehashed.  
To perform automated queries through scripts, you must create an account and obtain:

- **Dehashed username (email)**
    
- **API key**
    

These credentials are required for API authentication.

---

## Credential Discovery

When querying a target domain, Dehashed may return:

- Email addresses
    
- Usernames
    
- Cleartext passwords
    
- Hashed passwords
    
- Associated breach sources (databases)
    

Most discovered credentials are typically **old and may not work against externally exposed services or Active Directory authentication**. However, **password reuse remains common**, and valid credentials may occasionally be identified.

---

## Dehashed Query and Crack

The repository provides two main scripts.

### 1. dehashed.py

`dehashed.py` is a **Dehashed query tool** that searches for leaked credentials associated with a target domain. It outputs gathered **cleartext passwords and hashes** to files.

#### Usage

```bash
python3 dehashed.py [-a API_KEY] [-u USERNAME] -d DOMAIN [-o OUTPUT]
```

Options:

- `-a` → Dehashed API key
    
- `-u` → Dehashed account email
    
- `-d` → Target domain to query
    
- `-f` → Parse previously saved JSON query
    
- `-o` → Save discovered credentials to output files
    

Example query:

```bash
python3 dehashed.py -d domain.com -o output_file.txt -a API_KEY -u user@domain.com
```

Parse a previously saved query:

```bash
python3 dehashed.py -f previous_query.json
```

### 2. hash_crack.py

`hash_crack.py` attempts to **crack collected password hashes** using online hash cracking services.

#### Usage

```bash
python3 hash_crack.py -f HASH_FILE
```

If no file is specified, it will use the default output generated by `dehashed.py`.

Example:

```bash
python3 hash_crack.py -f
```

Or specify a custom hash file:

```bash
python3 hash_crack.py -f hash_file.txt
```

The script will attempt to crack hashes and return discovered passwords.
